# 搶票助手 (Ticket Tactical)

個人用的**合法**搶票輔助 Chrome / Edge 瀏覽器外掛(Manifest V3)。

它不會幫你自動買票 —— 它做的是把購票頁面變得更好操作:清掉干擾版面、放大關鍵按鈕、篩選票券、毫秒倒數校時、釋票提醒,以及解鎖後一鍵帶入實名個資(**填完不送出,送出永遠由你本人點**)。

## 支援站台

| 站台 | 網域 |
|------|------|
| 拓元 tixCraft | `tixcraft.com` |
| Weverse | `weverse.io` |
| YES24 Ticket | `ticket.yes24.com` |
| NOL World | `world.nol.com` |
| Melon Global | `tkglobal.melon.com` |

## 界線(設計原則,不可違反)

- **不自動送單**:外掛從不點擊「送出 / 購買 / 確認付款」,送出一律由你本人手動點。
- **不破解驗證碼、不多帳號掃票、不掛機**。
- **純本機**:除了對「售票站本站」做校時用的 HEAD 請求(只讀回應的 `Date` 標頭、不帶個資)外,不連任何後端、不蒐集遙測、不外傳任何資料。
- 個資以 **AES-GCM 加密 + 主密碼鎖** 保存,金鑰只存在記憶體;關閉所有售票分頁即自動上鎖。
- 站台改版導致 adapter 失效時,只會停用增強功能,**絕不阻擋原生購票流程**。

## 安裝

1. 下載或 clone 這個 repo
2. Chrome / Edge 開啟 `chrome://extensions`
3. 右上角開啟「**開發人員模式**」
4. 點「**載入未封裝項目**」→ 選擇 `extension/` 資料夾
5. 把外掛釘到工具列;開啟任一支援站台的頁面即生效

## 功能

- **站點偵測 + 清版面 + 放大關鍵按鈕**(tixCraft 選擇器已對照實際 DOM)
- **票券篩選**:關鍵字 / 黑名單 / 隱藏售完,支援 `逗號=AND`、`加號=OR`,數字自動正規化(`2800` ↔ `2,800`)
- **個資保險箱**:多筆實名個資,AES-GCM 加密、PBKDF2 主密碼派生金鑰;關閉所有售票分頁自動上鎖
- **一鍵填入**:解鎖後把個資帶入購票表單(heuristic 欄位對應),**填完不送出**
- **釋票監控**:輪詢偵測可購票 → WebAudio 嗶聲 + 桌面通知(不做任何自動操作)
- **倒數校時**:對售票站校時、毫秒級倒數浮動工具列(可收合成精簡膠囊)
- **自動刷新**:到點重整;在排隊 / 驗證 / 購票流程頁、或你正在打字時自動暫停(不會把你刷出隊伍)
- **Popup**:狀態總覽、解鎖、5 個功能開關、站台啟用、票券篩選規則、一鍵填入、監控啟停
- **Options**:實名個資 CRUD、主密碼管理(設定 / 解鎖 / 重設,重設會清空所有已存個資)

## 檔案結構

```
extension/
├─ manifest.json                 MV3;host 限支援站台,權限僅 storage + notifications
├─ lib/
│  ├─ messages.js                共享契約:訊息型別 / storage key / 預設值 / 欄位
│  ├─ crypto.js                  Vault 加解密(AES-GCM + PBKDF2)
│  ├─ filter.js                  票券篩選邏輯(AND/OR + 數字正規化)
│  └─ theme.css                  設計 token(popup / options 共用)
├─ content/
│  ├─ adapters.js                站台選擇器契約(tixCraft 已校準)
│  ├─ content.js                 注入:清版面/放大/篩選/工具列/校時/監控/自動刷新/一鍵填入
│  └─ content.css                注入樣式(tt- 前綴)
├─ background/service-worker.js  Vault 金鑰(記憶體)+ 訊息路由 + 通知 + 關分頁上鎖
├─ popup/                        工具列彈窗
├─ options/                      設定頁
└─ icons/                        icon 16 / 48 / 128
```

## 已知需微調處

- `content/adapters.js` 的 `keyButtonSelectors` 與 `soldOutSelector`:部分頁面的選擇器是推測值,站台改版或不同活動頁需自行校準。
- 一鍵填入的欄位對應目前靠關鍵字 heuristic(姓名 / 手機 / email / 證件 / 地址 / 生日),命中率依各站表單而異。
- 校時依賴售票站回應 `Date` 標頭;取不到時退回本機時間並標示「未校時」。
- 自動刷新與釋票監控**預設為關**,需在 popup 手動開啟。
